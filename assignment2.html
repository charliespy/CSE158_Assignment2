Section 1: Task Description



We will attempt to classify whether a given text was written by a human or generated by an AI/LLM. This is a binary classification task where:
- Input: Text samples from the Human vs LLM Text Corpus dataset (https://www.kaggle.com/datasets/starblasters8/human-vs-llm-text-corpus)
- Output: Binary label indicating whether the text is "Human" or "AI-generated"
- Task Type: Binary text classification
 

Model Evaluation



We will evaluate our models using accuracy as the primary metric. Accuracy measures the proportion of correctly classified texts out of all texts in the test set:

$$\text{Accuracy} = \frac{\text{Number of Correct Predictions}}{\text{Total Number of Predictions}}$$

The dataset will be split into training, validation, and test sets. We will:
1. Train models on the training set
2. Tune hyperparameters using the validation set
3. Report final performance metrics (accuracy, precision, recall, F1-score) on the held-out test set

Additionally, we will examine confusion matrices to understand the types of errors made by each model (e.g., false positives vs false negatives).
 

Relevant Baselines for Comparison



We will compare our models against the following baseline approaches from the course content:

1. Logistic Regression: A linear classifier that models the probability of a class using a logistic function. This is a fundamental baseline for binary classification tasks and works well with text features (e.g., TF-IDF vectors).

2. Support Vector Machine (SVM): A maximum-margin classifier that finds the optimal hyperplane to separate classes. SVMs are effective for text classification and can handle high-dimensional feature spaces well.

3. Naive Bayes: A probabilistic classifier based on Bayes' theorem with strong independence assumptions between features. Naive Bayes is particularly well-suited for text classification tasks and serves as a strong baseline.

4. Random Forest: An ensemble method that combines multiple decision trees. Random Forest can capture non-linear relationships and feature interactions, making it a robust baseline for comparison.

These baselines will be implemented using appropriate text feature representations (e.g., TF-IDF, bag-of-words) to enable fair comparison across models.
 

Assessing Model Prediction Validity



To assess the validity of our model's predictions, we will employ several strategies:

1. Train-Validation-Test Split: We will use a proper data split (e.g., 60-20-20 or 70-15-15) to ensure that model performance is evaluated on unseen data, preventing overfitting and providing realistic performance estimates.

2. Cross-Validation: For hyperparameter tuning and model selection, we will use k-fold cross-validation on the training set to get more robust performance estimates.

3. Error Analysis: We will examine misclassified examples to understand:
- Which types of texts are commonly misclassified (e.g., very short texts, technical writing, creative writing)
- Whether there are systematic biases in the predictions
- Patterns in false positives (human text classified as AI) vs false negatives (AI text classified as human)

4. Feature Analysis: We will investigate which features (e.g., text length, vocabulary diversity, sentence structure) are most predictive and whether they align with known characteristics of human vs AI-generated text.

5. Statistical Significance Testing: We will compare model performances using appropriate statistical tests to determine if improvements over baselines are statistically significant.

6. Confusion Matrix and Classification Metrics: Beyond accuracy, we will report precision, recall, and F1-score for each class to understand the trade-offs between different types of errors.

def lexical_diversity(text):
    words = str(text).split()
    if len(words) == 0:
        return 0
    return len(set(words)) / len(words)
data["lexical_diversity"] = data["text"].apply(lexical_diversity)

class TextDataset(Dataset):
    def __init__(self, X_sparse, y):
        self.X = torch.tensor(X_sparse.toarray(), dtype=torch.float32)
        self.y = torch.tensor(y, dtype=torch.float32)
    
    def __len__(self):
        return len(self.y)
    
    def __getitem__(self, idx):
        return self.X[idx], self.y[idx]

# Neural Network Model
class LLMDetector(nn.Module):
    def __init__(self, input_dim, hidden_dims=[512, 256, 128], dropout_rate=0.3):
        super(LLMDetector, self).__init__()
        
        layers = []
        prev_dim = input_dim
        
        for hidden_dim in hidden_dims:
            layers.extend([
                nn.Linear(prev_dim, hidden_dim),
                nn.BatchNorm1d(hidden_dim),
                nn.ReLU(),
                nn.Dropout(dropout_rate)
            ])
            prev_dim = hidden_dim
        
        layers.append(nn.Linear(prev_dim, 1))
        layers.append(nn.Sigmoid())
        
        self.network = nn.Sequential(*layers)
    
    def forward(self, x):
        return self.network(x).squeeze()

# Training function
def train_epoch(model, loader, criterion, optimizer, device):
    model.train()
    total_loss = 0
    correct = 0
    total = 0
    
    for X_batch, y_batch in loader:
        X_batch, y_batch = X_batch.to(device), y_batch.to(device)
        
        optimizer.zero_grad()
        outputs = model(X_batch)
        loss = criterion(outputs, y_batch)
        loss.backward()
        optimizer.step()
        
        total_loss += loss.item() * len(y_batch)
        predictions = (outputs >= 0.5).float()
        correct += (predictions == y_batch).sum().item()
        total += len(y_batch)
    
    return total_loss / total, correct / total

# Evaluation function
def evaluate(model, loader, criterion, device):
    model.eval()
    total_loss = 0
    correct = 0
    total = 0
    all_preds = []
    all_labels = []
    
    with torch.no_grad():
        for X_batch, y_batch in loader:
            X_batch, y_batch = X_batch.to(device), y_batch.to(device)
            outputs = model(X_batch)
            loss = criterion(outputs, y_batch)
            
            total_loss += loss.item() * len(y_batch)
            predictions = (outputs >= 0.5).float()
            correct += (predictions == y_batch).sum().item()
            total += len(y_batch)
            
            all_preds.extend(predictions.cpu().numpy())
            all_labels.extend(y_batch.cpu().numpy())
    
    return total_loss / total, correct / total, np.array(all_preds), np.array(all_labels)

for epoch in range(num_epochs):
    train_loss, train_acc = train_epoch(nn_model, train_loader, criterion, optimizer, device)
    val_loss, val_acc, _, _ = evaluate(nn_model, val_loader, criterion, device)
    
    train_losses.append(train_loss)
    val_losses.append(val_loss)
    train_accs.append(train_acc)
    val_accs.append(val_acc)
    
    # Learning rate scheduling
    scheduler.step(val_loss)
    
    # Early stopping check
    if val_loss < best_val_loss:
        best_val_loss = val_loss
        best_model_state = nn_model.state_dict().copy()
        patience_counter = 0
    else:
        patience_counter += 1
    
    print(f"Epoch {epoch+1:2d}/{num_epochs} | Train Loss: {train_loss:.4f} | Train Acc: {train_acc:.4f} | Val Loss: {val_loss:.4f} | Val Acc: {val_acc:.4f}")
    
    if patience_counter >= patience:
        print(f"\nEarly stopping triggered at epoch {epoch+1}")

# Load best model
nn_model.load_state_dict(best_model_state)

Section 5: Discussion

 

5.1 Summary of Results



Our experiments on the Human vs LLM Text Corpus dataset demonstrate that distinguishing between human-written and AI-generated text is a solvable classification problem with current machine learning techniques. The Neural Network achieved the best performance with 91.91% test accuracy, significantly outperforming all baseline models:

| Model | Test Accuracy | Test F1-Score |
|-------|--------------|---------------|
| Neural Network | 91.91% | 92.80% |
| SVM | 83.71% | 85.84% |
| Logistic Regression | 83.38% | 85.53% |
| Random Forest | 80.62% | 82.52% |
| Naive Bayes | 75.63% | 78.06% |

Key Findings:

1. Deep learning superiority: The Neural Network achieved an 8-point improvement in accuracy over the next best model (SVM), demonstrating that non-linear feature interactions captured by deep networks are crucial for this task.

2. TF-IDF features are highly effective: Models using TF-IDF representations (Logistic Regression, SVM, Neural Network) consistently outperformed the Random Forest model that relied solely on engineered features, suggesting that lexical patterns are strong discriminators.

3. Generalization gap: The Neural Network showed the largest gap between training (99.79%) and test accuracy (91.91%), indicating some overfitting. However, the test performance remains robust and significantly better than baselines.

4. Consistent precision-recall trade-off: All models showed higher recall than precision for detecting AI-generated text, meaning they are slightly more likely to misclassify human text as AI than vice versa.
 

5.2 Discussion of Related Work



How has this dataset (or similar datasets) been used before?

Similar AI-text detection datasets have been used extensively in recent research. The Ghostbuster system (Verma et al., 2023) used student essays and news articles to achieve F1 scores of 99.0. The RAID Benchmark has become a standard for evaluating detection models, with recent methods like TDT achieving AUROC of 0.855. Our dataset (788,922 samples) is larger than most benchmarks, providing robust training data.

How has prior work approached the same task?

Prior approaches fall into three main categories:
- Zero-shot methods (e.g., DetectGPT): Analyze probability distributions without training, exploiting AI text's lower perplexity
- Supervised classifiers: Fine-tuned transformers (RoBERTa, GPTZero) typically achieve 90-95% accuracy; TF-IDF with traditional ML achieves 80-85%
- Hybrid methods (e.g., Ghostbuster): Combine LM-based features with classifiers for state-of-the-art results

How do our results compare?

| Method | Performance | Comparison |
|--------|-------------|------------|
| Ghostbuster (SOTA) | F1 99.0 | Our NN: F1 92.80 (gap due to simpler features) |
| Fine-tuned RoBERTa | 90-95% acc | Our NN: 91.91% (competitive) |
| TF-IDF + SVM baselines | 80-85% acc | Our SVM: 83.71% (matches literature) |

Our Neural Network achieves competitive performance (91.91% accuracy) using simpler TF-IDF features rather than transformer embeddings, validating that traditional feature engineering remains effective for this task.
 

5.3 Feature Analysis



Key discriminative features identified:
- Text length: Human texts are longer (mean: 710 words vs 331) with higher variance, while AI texts are more uniform
- Lexical diversity: Counterintuitively, AI texts show higher diversity (0.628 vs 0.561), suggesting LLMs avoid repetition
- TF-IDF patterns: The strong performance of TF-IDF models indicates vocabulary usage and phrase structures differ significantly between classes
 

5.4 Error Analysis



The Neural Network shows balanced error rates (~6.6% for both classes). Common misclassification patterns include:
- Short texts (less discriminative information)
- Formal human writing misclassified as AI
- Conversational AI text misclassified as human
 

5.5 Limitations



- Generalization: Models trained on this dataset may not generalize to other domains or newer LLMs
- Feature constraints: TF-IDF limited to 5,000 features; more sophisticated representations (transformer embeddings, perplexity) not explored
- Adversarial robustness: Paraphrasing or human editing of AI text could easily fool these detectors
 

5.6 Future Work



- Use transformer embeddings (BERT, RoBERTa) for richer feature representations
- Fine-tune transformer-based classifiers for potentially higher accuracy
- Include adversarial training with paraphrased text for robustness
- Evaluate on multi-source AI text (GPT-4, Claude, Llama) for generalization
 

5.7 Conclusion



We demonstrated that AI-generated text detection achieves high accuracy using machine learning. Our Neural Network with TF-IDF features achieved 91.91% test accuracy (F1: 92.80%), outperforming traditional baselines by 8+ points. Key discriminative features include text length, lexical diversity, and vocabulary patterns. While effective, detection methods must continuously adapt as LLMs evolve.
